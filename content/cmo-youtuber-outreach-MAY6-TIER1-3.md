Hey Chris / CSS-Tricks editorial team,

I built a privacy-first YouTube analytics dashboard called Viewpulse (viewpulse.rachkovan.com) and I think the technical approach would make for an interesting CSS-Tricks article.

**The tech stack:**
- Next.js 14 (App Router)
- TypeScript + Tailwind CSS + shadcn/ui
- Recharts for data visualization
- JSZip for client-side ZIP parsing
- YouTube Data API v3 (optional OAuth path)

**The interesting technical angle:**
The entire application processes analytics data client-side. Users upload their YouTube watch-history.html (or connect via OAuth), and all parsing, aggregation, and visualization happens in the browser. No backend data processing. No data collection. Privacy enforced by architecture, not policy.

This required some creative approaches: chunked ZIP parsing for large files, aggressive memoization for repeated calculations, and a completely stateless backend.

**Potential article angles:**
1. "Building a client-side analytics dashboard with Next.js and Recharts"
2. "Privacy-first web apps: architecture patterns that keep data in the browser"
3. "Parsing YouTube Takeout data with JavaScript: a case study"
