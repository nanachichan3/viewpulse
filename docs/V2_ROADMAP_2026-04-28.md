# Viewpulse v2 — Technical Roadmap

**Date:** 2026-04-28
**Author:** CTO
**Status:** Draft for CEO Review
**Project:** Viewpulse (Project 28)

---

## 1. Overview

Viewpulse v2 expands the privacy-first YouTube watch history analyzer from a single-file upload tool into a richer analytics platform for power users and creators.

**Core constraint:** All v2 features must remain client-side first. No user data leaves the browser unless explicitly opted in.

---

## 2. v2 Feature Scope

### F1: Video Comparison (Side-by-Side Analytics)
**User Story:** As a creator, I want to compare two YouTube videos' performance metrics side-by-side.

**Technical Approach:**
- Use YouTube Data API v3 for public video metadata (views, likes, comments, upload date)
- No OAuth required for public data — just an API key
- Client-side caching in IndexedDB to avoid redundant API calls
- Compare: views, engagement rate, like ratio, comment sentiment, upload frequency

**API Requirements:**
- `GET https://www.googleapis.com/youtube/v3/videos?part=statistics,snippet&id={videoId}&key={API_KEY}`
- Rate limit: 10,000 quota units/day per key
- Cost per video: ~3 quota units

**Effort:** 12h (UI + API client + caching)

### F2: Trend Analysis
**User Story:** As a user, I want to see how my watch habits change over time with trend detection.

**Technical Approach:**
- Extend existing `AnalyticsResult` with delta calculations
- Month-over-month and year-over-year comparisons
- Detect emerging interests (new channel appearances, genre shifts)
- Highlight "binge weeks" and "dry spells"

**Data Model Extensions:**
```typescript
interface TrendAnalysis {
  periodStart: Date;
  periodEnd: Date;
  watchCountDelta: number;      // vs previous period
  uniqueChannelsDelta: number;
  newChannels: string[];        // channels not seen in previous period
  droppedChannels: string[];    // channels from previous period not seen now
  topGrowingChannel: { name: string; growthPct: number };
  peakBingeDay: { date: string; count: number };
  avgSessionLength: number;     // estimated minutes between videos
}
```

**Effort:** 8h (analytics engine + charts)

### F3: Alert Thresholds
**User Story:** As a user, I want to set personal goals and get notified when I hit watch-time milestones.

**Technical Approach:**
- LocalStorage-based goal settings (no backend)
- Thresholds: max daily watch count, max consecutive hours, channel-specific limits
- Visual alerts in dashboard (no push notifications — fully client-side)
- Weekly summary report generated in-browser

**Effort:** 6h (settings UI + alert logic)

### F4: Enhanced Export
**User Story:** As a user, I want to export my analytics in multiple formats.

**Technical Approach:**
- JSON export (raw data)
- CSV export (tabular data for spreadsheet analysis)
- PDF report (themed, printable)
- Image export (charts as PNG)

**Libraries:**
- `json2csv` for CSV generation
- `html2canvas` + `jspdf` for PDF
- Recharts built-in PNG export

**Effort:** 10h

### F5: Creator Mode (Opt-in)
**User Story:** As a YouTube creator, I want to connect my channel and see audience analytics.

**Technical Approach:**
- OAuth 2.0 with YouTube Data API (separate from watch history)
- Requires `https://www.googleapis.com/auth/youtube.readonly` scope
- Serverless proxy (Vercel API route) to protect API key
- Display: subscriber trends, video performance, audience retention

**Security:**
- OAuth tokens stored in httpOnly cookies (not localStorage)
- Token refresh handled server-side
- User can revoke access anytime

**Effort:** 16h (OAuth flow + API proxy + creator dashboard)

## 4. Technical Debt & Refactoring

### R1: State Management
Current state is prop-drilled through Dashboard. v2 should use:
- Zustand for global analytics state
- Persist to localStorage for session recovery

### R2: Parser Performance
For users with 50k+ watch events:
- Move parsing to Web Worker
- Add progress indicator for large files
- Stream processing for files >10MB

### R3: Chart Optimization
- Virtualize large datasets in Recharts
- Add data aggregation controls (daily/weekly/monthly)

**Effort:** 8h
